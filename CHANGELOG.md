## [2.1.1] - 2026-04-22

### 修复
- **修复 `/角色列表` 指令报错**：原代码使用 `search_memory(query_text="", ...)` 查询角色列表，空字符串导致 Embedding API 返回 `"input.texts should not be null"` 错误
  - 新增 `query_memories` 方法，使用 Milvus `collection.query()` 直接基于过滤表达式查询记录，无需调用 Embedding API
  - `list_known_characters` 方法改用 `query_memories` 替代 `search_memory`
- **防御性检查**：在 `get_embedding` 方法入口增加空文本检查，防止空字符串被发送到 Embedding API

## [2.1.0] - 2026-04-21

### 新增功能
- **对话历史存入**：将图片识别结果和AI回复存入对话历史，解决AI不知道自己看过图片导致对话跳跃的问题
  - 识别到自己时：存入用户消息（包含图片描述和自身特征）和AI回复
  - 识别到角色时：存入用户消息（包含图片描述和角色特征）和AI回复
  - 常规识图时：存入用户消息（包含图片内容描述）和AI回复
  - 使用 `conversation_manager.add_message_pair()` 方法将消息对写入当前对话历史
  - 后续对话中AI可以回顾之前看过什么图片、识别了什么角色，保持对话连贯性

---

## [2.0.2] - 2026-04-20

### 新增功能
- **语言风格保持**：在所有回复生成时，明确要求 LLM 保持与对话上下文一致的语言风格、格式和用语习惯
  - 自动学习并延续原有的说话方式（如括号描述动作、特殊表达习惯等）
  - 保持相同的句式结构、语气词和表达习惯
  - 让回复自然融入之前的对话风格中，提供更连贯的对话体验

### 改进
- **优化 system_prompt 设计**：在 `main.py`、`self_recognition.py` 和 `character_recognition.py` 的所有回复生成函数中增强提示词
  - 新增第4条关键要求：保持语言风格一致性
  - 详细说明需要保持的语言特征类型
  - 强调根据对话上下文自然融入回复风格

## [2.0.1] - 2026-04-20

### 新增功能
- **增强特征识别**：新增眼白颜色、眼瞳颜色、种族特征的识别和对比
  - 眼白颜色：通常为白色，特殊角色可能有其他颜色
  - 眼瞳颜色：虹膜颜色，支持异色瞳描述
  - 种族特征：猫耳、龙角、尾巴、翅膀等非人类特征
- **种族特征相似度计算**：新增 `calculate_feature_similarity` 方法，支持多种族特征对比
- **独立阈值配置**：各项特征相似度阈值可在 WebUI 独立配置
  - `hair_color_threshold` - 发色相似度阈值
  - `eye_white_threshold` - 眼白颜色相似度阈值
  - `eye_pupil_threshold` - 眼瞳颜色相似度阈值
  - `racial_feature_threshold` - 种族特征相似度阈值

### 改进
- **图片消息处理优化**：模拟 AI 拥有视觉能力
  - 将"用户发了一个图片过来"改为"我看到了..."
  - 以第一人称视角自然地回应图片内容
- **上下文回顾功能**：自动获取最近 5 条对话历史
  - 确定当前话题和语境
  - 参照上下文生成更自然的回复
- **视觉模型提示词优化**：使用结构化格式提取特征，确保特征提取的完整性和一致性
- **特征对比逻辑增强**：从原来的发色+瞳色两项对比，扩展为发色+眼白+眼瞳+种族特征四项对比
- **兼容旧格式**：支持旧版本存储的特征数据，自动兼容 `eye_color` 字段
- 优化 system_prompt 设计，让回复更加拟人化

### 修复
- **修复重复日志问题**：优化图片识别流程，在 main.py 中提前进行人物检测和特征提取，避免 self_recognition 和 character_recognition 模块重复调用视觉模型导致的重复日志
- **修复颜色特征缓存丢失问题**：当颜色特征缓存为空时，自动从数据库存储的记忆文本中提取颜色特征，解决插件重启后所有图片都被误识别为自己的问题
- **删除多余的日志输出**：移除 `generate_self_response`、`generate_character_response` 和 `_handle_normal_image_conversation` 中的重复日志，避免日志污染

## [2.0.0] - 2026-04-20

### 重构
- **模块化重构**：将插件拆分为多个独立模块，便于维护和扩展
  - `modules/milvus_manager.py` - Milvus 数据库管理模块
  - `modules/image_processor.py` - 图片处理和特征提取模块
  - `modules/self_recognition.py` - 认识自己功能模块
  - `modules/character_recognition.py` - 认识角色功能模块（新增）

### 新增功能
- **认识角色功能**：使用 `/认识角色` 指令，让 AI 记住其他角色的形象
- **角色列表指令**：使用 `/角色列表` 查看已认识的所有角色
- **数据库字段扩展**：新增 `character_type` 和 `character_name` 字段，区分自己和其他角色

### 修复
- **修复数据库连接配置错误**：原代码使用 `milvus_db_id` 读取配置，现已修正为 `milvus_db_name`，确保正确连接到 WebUI 配置的指定数据库

### 改进
- **自动识别优先级**：图片识别时优先检查是否是自己，其次检查是否是已认识的角色
- **角色颜色特征缓存**：为每个角色单独缓存颜色特征，提高识别准确性
- **日志信息优化**：统一使用 `[角色认知]` 作为日志前缀

### 指令变更
- `/认识自己` - 教 AI 认识自己的形象（保持不变）
- `/认识角色` - 教 AI 认识其他角色的形象（新增）
- `/角色列表` - 查看已认识的角色列表（新增）
- `/角色认知设置` - 查看当前配置（原 `/自我认知设置`）
